Help

fpcurator is a collection of tools for performing certain curation tasks easier. There are four sub-tools available. Note that the tool may freeze while performing any task. All debug info on any task is printed to the console the program should have launched with.

Download URLs

The Download URLs tool downloads and formats a list of files from a list of urls into a collection of organized folders inside the directory specified by "Output Folder". It works in a similar way to cURLsDownloader, but is powered by fpclib. Put all the urls you want to download into the textbox and press "Download".
Here are a list of options:

  • Remove "web.archive.org" - When checked, the downloader will put all urls downloaded from the web archive back into their original domains.
  • Keep URLVars - When checked, the downloader will append url vars present on links being downloaded to the end of the html file. This is only necessary when you have two links to the same webpage that generate different html due to the url vars.
  • Clear URLs - When checked, the downloader will clear any urls in the list when they are downloaded. Errored urls will remain in the list.
  • Show Done - When checked, the downloader will show a message box when it is done downloading.

Auto Curator

The Auto Curator tool generates a bunch of partial curations based upon a list of urls containing the games to curate. The list of curatable websites is limited the site definitions in the "sites" folder next to the program.
Here are a list of options:

  • Save - When checked, the auto curator will save it's progress so if it fails from an error, the tool will resume where it left off given the same urls.
  • Use Titles - When checked, the auto curator will use the titles of curated games instead of a randomly generated uuid.
  • Clear URLs - When checked, the auto curator will clear any urls in the list when they are curated. Errored urls will remain in the list.
  • Show Done - When checked, the auto curator will show a message box when it is done curating.

Search

The Search tool can take a list of game titles and a list of urls for those titles and bulk check how likely it is those games are in Flashpoint. You will need to load the Flashpoint database (Flashpoint/Data/flashpoint.sqlite) for it to check through, in addition to providing regexes that will be used to check if games in Flashpoint have the right Source, Publisher, or Developer that would match a game title in the list.
Here are a list of options:

  • Priorities - When checked, the searcher will generate a list of numeric priorities and print them into a file named "priorities.tsv" next to the program.
  • Log - When checked, the searcher will generate a more human readable log displaying how likely each it is those games are in Flashpoint.
  • Strip Subtitles - When checked, titles will be stripped of subtitles when searching for them in Flashpoint.
  • Exact Url Matches - When unchecked, the searcher will skip checking for exact url matches for a game match in Flashpoint. Normally an exact url match is a very good indicator if a game is curated, but this is optional in case multiple games are on the same url.
  • Use difflib - When checked, the searcher will use the default python difflib instead of the fast and efficient python-Levenshtein.

Wiki Data

The Wiki Data tool provides you with the ability to get a list of all Tags, Platforms, Games, or Animations in Flashpoint. Just select the given page and press "Find".
